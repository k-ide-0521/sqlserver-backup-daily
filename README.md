# SQLServerBackup

## 概要

製造現場から送信されたデータは、SQL Server上の複数データベースに蓄積している。  
毎日深夜に全データベースのフルバックアップを取得している。  
バックアップは曜日別にファイルを上書きすることで、直近7日分を常に保持する運用としている。

## バックアップ方針について

毎日フルバックアップを取得し、曜日をファイル名に含めることで7世代分を自動的に保持する。  
バックアップ完了後はバックアップセットの整合性検証（`RESTORE VERIFYONLY`）を実施し、取得したバックアップが復元可能な状態かを確認している。  
バックアップファイルは、バックアップ終了後に別デバイス（NAS）にコピーすることで、サーバー障害時にも復元できる体制を維持している。
